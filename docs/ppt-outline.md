# PPT 制作大纲

建议制作 10-12 页，答辩时间控制在 5-8 分钟。

## 1. 标题页

- 标题：多模态心理咨询辅助系统
- 副标题：基于视频、语音和大语言模型的课程设计原型
- 放置成员姓名、课程名称、日期。

## 2. 背景与问题

- 心理咨询需要综合语言、表情、语音和长期历史。
- 单一文本或单次访谈记录难以完整整理线索。
- 目标不是自动诊断，而是帮助用户和咨询师整理辅助信息。

建议截图：无，使用简洁问题陈述即可。

## 3. 系统目标

- 普通用户上传交流视频，获得非诊断性反馈。
- 心理咨询师查看授权用户历史。
- 系统生成咨询师辅助建议草稿。
- 保留报告、历史和备注，便于复核。

建议截图：首页或登录页。

## 4. 总体架构

- React + Vite 前端。
- FastAPI 后端。
- fastapi-users 认证。
- PostgreSQL + Alembic 数据持久化。
- DeepFace、SenseVoice、LLM 模型服务。

建议截图：`docs/architecture.md` 中的 Mermaid 流程图。

## 5. 普通用户流程

- 登录。
- 上传视频。
- 查看任务阶段。
- 查看报告和历史。
- 导出报告。

建议截图：普通用户上传与历史页。

## 6. 多模态分析流程

- 抽帧。
- 提取音频。
- 表情概率分析。
- 语音转写和声学特征。
- 多模态融合。
- LLM 生成建议。

建议截图：任务状态进度和报告摘要。

## 7. 报告展示

- 主情绪。
- 置信度。
- 风险等级。
- 证据列表。
- 模型名、prompt 版本和生成时间。
- 非诊断性声明。

建议截图：报告详情页。

## 8. 心理咨询师工作台

- 绑定普通用户。
- 查看用户历史。
- 生成辅助建议。
- 添加咨询备注。
- 查看趋势摘要。

建议截图：咨询师工作台。

## 9. 心理科普模块

- 展示常见心理问题基础知识。
- 覆盖抑郁、焦虑、睡眠、压力适应、创伤后应激、强迫相关问题。
- 强调科普不等于自我诊断。
- 提示需要尽快求助的情况。

建议截图：心理科普页。

## 10. 权限与安全边界

- 普通用户只能查看自己的任务和报告。
- 咨询师只能查看已绑定用户。
- `.env` 不提交真实 API Key。
- 报告输出坚持非诊断性。

建议截图：可使用接口分组表或权限说明。

## 11. 测试与运行

- `pytest` 后端测试。
- `npm run build` 前端构建。
- `docker compose config --quiet` 配置验证。
- Docker Compose 启动五个服务。
- 使用演示账号跑完整流程。

建议截图：Docker Compose 服务状态。

## 12. 总结与未来工作

- 已完成端到端 MVP。
- 项目亮点：多模态、角色权限、历史记录、咨询师辅助、科普模块。
- 局限：非临床验证、样本少、任务队列仍为课程级。
- 未来：任务队列、审计日志、数据删除、趋势图、更多评估样例。
