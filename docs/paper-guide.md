# 小论文写作指南

本文档面向课程设计小论文撰写人员，用于快速理解项目定位、技术路线和可写内容。

## 题目建议

- 基于多模态视频分析的心理咨询辅助系统设计与实现
- 面向心理咨询场景的多模态情绪分析与辅助建议生成系统
- 融合表情、语音与大语言模型的心理健康辅助原型系统

## 摘要写作要点

摘要可按“四句话结构”组织：

1. 说明背景：心理咨询场景中，咨询师需要综合观察用户表情、语音和语言内容。
2. 说明方法：系统使用 React、FastAPI、PostgreSQL、DeepFace、SenseVoice 和大语言模型构建多模态分析流程。
3. 说明功能：普通用户上传交流视频获得非诊断性建议，心理咨询师查看授权用户历史并生成辅助草稿。
4. 说明边界：系统定位为课程设计原型和专业辅助工具，不能替代诊断、治疗或危机干预。

## 研究背景

可从以下角度展开：

- 心理咨询依赖访谈、观察和持续跟踪，单次文字描述容易遗漏非语言线索。
- 视频中包含表情、语音、停顿、语速等信息，多模态分析有助于整理咨询线索。
- 大语言模型可以把结构化特征转化为更易阅读的建议草稿，但需要明确非诊断边界。
- 课程设计重点不是临床诊断，而是验证“多模态分析 + 角色权限 + 历史记录 + 辅助建议”的工程流程。

## 系统设计

系统采用前后端分离和模型服务解耦架构：

- 前端：React + Vite，提供登录、上传、任务状态、报告、咨询师工作台和心理科普页面。
- 后端：FastAPI，负责任务编排、权限校验、报告生成、文件保存和接口输出。
- 认证：fastapi-users 提供注册、JWT 登录和当前用户能力。
- 数据库：PostgreSQL 保存用户、绑定关系、任务、报告和咨询备注；Alembic 管理表结构。
- 模型服务：DeepFace 负责人脸表情分析，SenseVoice 负责语音转写和语音侧信息。
- 专家系统：使用 OpenAI 兼容接口生成普通用户建议和咨询师辅助草稿。

## 核心功能

普通用户端：

- 使用邮箱和密码登录。
- 上传交流视频。
- 查看任务进度、失败原因和历史记录。
- 查看主情绪、置信度、风险等级、人脸表情、语音特征和专家意见。
- 导出 JSON 或文本报告。
- 查看已授权心理咨询师。

心理咨询师端：

- 绑定普通用户邮箱。
- 查看已绑定用户列表。
- 查看单个用户多次分析历史。
- 生成咨询师辅助建议草稿。
- 添加人工咨询备注。
- 查看用户情绪变化趋势摘要。

科普模块：

- 展示抑郁、焦虑、睡眠、压力适应、创伤后应激和强迫相关问题的基础知识。
- 明确内容仅用于心理健康教育，不用于自我诊断。
- 提示出现自伤/伤人想法、严重失眠或功能受损时应尽快求助。

## 前端界面设计与用户体验

前端界面围绕“可信、清晰、可演示”设计：

- 首页使用温和的心理咨询辅助主视觉图，避免医疗急救或夸张痛苦符号。
- 顶部导航分为“工作台、心理科普、项目说明”，让答辩时可以快速切换模块。
- 普通用户工作台突出上传入口、任务状态和历史报告。
- 咨询师工作台突出绑定用户、历史查看、辅助建议、备注和趋势。
- 报告页展示模型名、prompt 版本、生成时间和风险依据，便于说明可追溯性。
- 失败任务会显示阶段和错误原因，避免演示时只看到空白结果。

## 技术路线

视频分析流程：

1. 用户上传视频。
2. 后端保存原始视频并创建任务。
3. 后端抽取关键帧和音频。
4. DeepFace 分析关键帧表情概率。
5. SenseVoice 转写语音并提取语音侧信息。
6. 融合模块生成主情绪、置信度、风险等级和证据。
7. LLM 根据结构化结果生成非诊断性建议。
8. 报告写入数据库，并保留本地 JSON 文件。
9. 普通用户和授权咨询师按权限查看结果。

## 实验与测试说明

课程论文中可以把“系统测试”写成工程验证，而不是医学效果验证：

- 功能测试：登录、上传、任务状态、报告查看、咨询师绑定、辅助建议、备注、导出。
- 权限测试：普通用户不能查看他人报告；咨询师只能查看已绑定用户。
- 构建测试：`pytest`、`npm run build`、`docker compose config --quiet`。
- 部署测试：Docker Compose 启动 PostgreSQL、backend、frontend、DeepFace、SenseVoice。
- 演示测试：使用演示账号完成普通用户和咨询师两条流程。

## 伦理边界

论文必须明确：

- 系统是心理咨询辅助工具，不是自动诊断系统。
- 风险等级只表示复核优先级，不等于疾病判断。
- LLM 输出只能作为沟通草稿，最终判断由专业人员负责。
- 项目不提供危机干预能力，紧急情况应联系专业机构或当地紧急服务。
- 演示视频应避免使用真实隐私数据。

## 局限性

可以写入论文的局限：

- 演示样本规模小，不能证明临床有效性。
- 表情和语音模型可能受光照、角度、噪声、语言风格影响。
- 当前任务队列使用 FastAPI BackgroundTasks，适合课程演示，但不是生产级可靠队列。
- 文件保留和数据删除策略仍较简单。
- 风险等级规则较基础，缺少专业量表和人工标注数据验证。

## 未来工作

- 增加审计日志、数据删除接口和文件生命周期策略。
- 增加咨询师人工复核状态和更完整的备注体系。
- 扩展趋势图和多次咨询对比分析。
