# 多模态心理咨询辅助系统部署实施方案

## 1. 文档信息

- 文档版本：v1.0
- 编写日期：2026-06-14
- 部署方式：Docker Compose 为主，本地 Conda 开发为辅

## 2. 部署目标

在本地或演示服务器上部署完整系统，包含前端、后端、DeepFace 表情服务、SenseVoice 语音服务和 PostgreSQL 数据库，使普通用户和心理咨询师流程能够完整运行。

## 3. 环境准备

### 3.1 基础软件

| 软件 | 用途 |
| --- | --- |
| Docker Desktop / Docker Engine | 容器运行环境 |
| Git | 拉取和管理代码 |
| PowerShell | Windows 脚本执行 |
| 浏览器 | 访问前端和接口文档 |
| FFmpeg | 后端容器内用于抽帧和音频提取 |

### 3.2 端口规划

| 端口 | 服务 |
| --- | --- |
| 5173 | 前端 Vite 服务 |
| 8000 | FastAPI 后端 |
| 8001 | DeepFace 服务 |
| 8002 | SenseVoice 服务 |
| 5432 | PostgreSQL |

如端口冲突，应修改 `docker-compose.yml` 中对应端口映射。

## 4. 配置文件

首次部署建议复制环境变量文件：

```powershell
Copy-Item .env.example .env
```

关键配置：

| 配置项 | 说明 |
| --- | --- |
| `OPENAI_BASE_URL` | OpenAI 兼容接口地址 |
| `OPENAI_API_KEY` | LLM API Key，不配置时使用兜底建议 |
| `OPENAI_MODEL` | LLM 模型名 |
| `AUTH_SECRET_KEY` | JWT 签名密钥，生产环境必须修改 |
| `BACKEND_CORS_ORIGINS` | 前端来源 |
| `DATABASE_URL` | PostgreSQL 连接串 |

## 5. Docker Compose 部署

### 5.1 构建并启动

```powershell
docker compose up --build
```

### 5.2 后台启动

```powershell
docker compose up -d --build
```

### 5.3 使用已有镜像启动

```powershell
docker compose up -d --no-build
```

### 5.4 查看服务状态

```powershell
docker compose ps
```

### 5.5 查看日志

```powershell
docker compose logs -f backend
docker compose logs -f deepface
docker compose logs -f sensevoice
```

## 6. 数据库初始化

Docker Compose 中 PostgreSQL 启动并健康检查通过后，后端容器执行 Alembic 迁移：

```powershell
alembic upgrade head
```

系统首次启动会创建演示账号：

| 角色 | 邮箱 | 密码 |
| --- | --- | --- |
| 普通用户 | `client@example.com` | `client123` |
| 心理咨询师 | `counselor@example.com` | `counselor123` |

## 10. 存储目录

| 目录 | 说明 |
| --- | --- |
| `storage/uploads` | 上传视频 |
| `storage/frames/{task_id}` | 抽帧图片 |
| `storage/audio/{task_id}.wav` | 提取音频 |
| `storage/reports/{task_id}.json` | 兼容报告 JSON |

这些目录为运行时数据，不应提交到 Git 仓库。

## 11. 部署后验证

1. 打开 `http://localhost:5173`。
2. 使用普通用户账号登录。
3. 上传一段视频，确认任务进度变化。
4. 查看报告详情并导出 JSON。
5. 退出后使用咨询师账号登录。
6. 绑定普通用户邮箱。
7. 查看用户历史、生成辅助建议、添加备注。
8. 访问 `http://localhost:8000/docs` 查看接口文档。

## 12. 回滚方案

### 12.1 代码回滚

```powershell
git log --oneline
git revert <commit>
```

### 12.2 容器回滚

如已保留旧镜像，可切换镜像标签后重新启动：

```powershell
docker compose down
docker compose up -d --no-build
```

### 12.3 数据回滚

课程演示环境可通过删除 Docker volume 重置数据库：

```powershell
docker compose down -v
docker compose up -d --build
```

注意：该操作会清空数据库和模型缓存，仅适用于演示环境。
