# Prompt 版本说明

当前系统使用轻量 prompt 版本管理，主要服务于课程答辩和报告可追溯。

## 当前版本

- `prompt_version`: `v1`
- `model_name`: 来自后端环境变量 `OPENAI_MODEL`
- `generated_at`: 报告生成时间

## v1 设计目标

- 根据视频摘要、表情分析、语音特征和综合预判生成非诊断性建议。
- 输出面向普通用户时保持谨慎、安抚和可执行。
- 输出面向咨询师时强调“仅供专业人员参考”，帮助整理访谈关注点。

## 为什么记录版本

- 方便说明报告使用了哪一版提示词。
- 后续修改 prompt 后，可以比较不同版本输出差异。
- 小论文可把它作为“可追溯性设计”的一部分。

## 后续比较方式

后续可准备固定样例报告，分别用 `v1`、`v2` 生成建议，然后比较：

- 是否保持非诊断性边界。
- 建议是否具体、温和、可执行。
- 是否准确引用输入证据。
- 是否避免夸大风险或替代专业判断。
